# 客户经营点击分析分页与汇总口径

## 背景

`客户经营点击分析` 用于在业务概览中查看 HTML 名单带来的客户经理点击行为。本次改造解决名单明细只能展示前 100 条的问题，并明确顶部统计和分页列表的口径边界。

## 术语口径

- `名单数`：当前筛选条件下，按 `list_key` 去重后的名单份数。
- `被点击名单数`：当前筛选条件下，`total_click_count > 0` 的名单份数。
- `名单点击率`：`被点击名单数 / 名单数`。
- `名单总客户数`：当前筛选条件下，所有名单客户数汇总。
- `被点击客户数`：当前筛选条件下，至少发生过一次洞察、电访或查看方案点击的客户数汇总。
- `客户点击覆盖率`：`被点击客户数 / 名单总客户数`。

## 展示规则

- 顶部指标使用当前筛选条件下的全量汇总，不随名单明细分页变化。
- 名单明细支持分页完整查看所有名单。
- 名单明细默认按点击总数倒序、最近点击时间倒序。
- 页面默认每页 20 条，可选 20、50、100。
- 客户点击明细本轮不做全量分页，仍随选中名单查看；选择全部名单时保持有限展示。

## 接口规则

增强现有 `/html-preview/lists`，不新增接口。

新增查询参数：

- `page`：页码，默认 1。
- `page_size`：每页条数，最大 100。页面调用显式传 20；无分页参数的旧调用默认保持 100。

兼容旧参数：

- `limit`：旧调用仍可使用，映射为第一页的 `page_size`，最大按 100 处理。

新增响应字段：

- `total`：全量名单数。
- `clicked_list_count`：全量被点击名单数。
- `page`：当前页码。
- `page_size`：当前页大小。
- `summary`：当前筛选条件下的全量汇总指标。
- `items`：当前页名单明细。
